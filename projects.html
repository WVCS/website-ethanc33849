My Projects

Scratch App

This "cat chases mouse" game was my first coding project in AP CSP. At first, the premise seemed quite simple—the user drags the mouse around while the cat chases it—but the project quickly became more and more complicated. Starting with my dislike of the default score/lives display, this "'Escape the Cat" project quickly marked the beginning of my dislike of Scratch, and more recently, code.org. To have a better-looking display for the user's lives, I was forced to create a complicated character that receives broadcasts and changes costumes (I had to create an absurd number of costumes, as I couldn't find a way to display text without having a character with a text bubble), all while keeping track of the score. Then, the issue of the cat looking horrible came into play. I manually created another costume for the cat to look more like it was running, then did the same for the mouse. As I spent more and more time working on the way the project looked, I quickly realized that my backdrops just didn't look that great. I couldn't find anything good on Google, so I quickly went into Blender and created a simple wood floor.

Moving on to the sprite mechanics, I wasn't happy with how the chasing felt quite unrealistic. There was no change in speed, and the game was either too easy or too hard. In the end, I decided to write some code that changed the speed of the cat depending on the position of the mouse, also adding cat "pounces" to the mouse. This introduced a whole new set of bugs, such as the cat being able to take multiple lives in one pounce. Therefore, I developed a system that created "I-frames", or a time when the mouse was safe and unable to be attacked. After adding a second level, I thought it would be great if the second level was substantially harder. I added a level system as a result, increasing the cat's speed but also decreasing the size of the cat and mouse—making the game as a whole faster and slightly more difficult.

While this game led to a lot of headaches, struggling with weird bugs and having to find workarounds that took up tens of lines of code, I'm still proud of this. This "Escape the Cat" game is still, to this day, the best game on Scratch that I've ever made—while there are things that I would change looking back in hindsight, I don't have the motivation to change much about it.
